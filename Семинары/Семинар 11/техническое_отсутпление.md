**Техническое отступление**

Данные библиотеки необходимо сначала устанавливать (можно через pip / conda или brew, если Вы работаете на MAC OS).
Чтобы у Вас точно вопроизводился ноутбук и не было проблем из-за несовпадающих версий библиотек, рекомендуется через python создавать виртуальную среду через командную строку:

```python
python3.8 -m venv my_venv
source my_venv/bin/activate
pip install -r requirements.txt
pip install ipykernel
python3 -m ipykernel install --user --name venv --display-name 'boosting_env'
```

Внутри файлика ```requirements.txt```, которые Вам нужно создавать каждый раз под проект, можете написать все версии библиотек, которые необходимо установить и использовать при активации виртуальной среды (т.е. деактивировав виртуальную среду, jupyter ноутбуки снова будут использовать текущие версии библиотек). 

Пример: недавно Вы установили библиотеку sklearn и скачали последнюю версию 1.0.1. Вам нужно запустить чужой ноутбук, и у автора этого ноутбука была установлена версия sklearn 0.15. В коде автор использует  ```from sklearn.cross_validation import train_test_split``` но у Вас возникает ошибка при импорте: ```ModuleNotFoundError: No module named 'sklearn.cross_validation'```, потому что в новой версии sklearn такого модуля уже нет, и фунция ```train_test_split``` переехала в модуль `sklearn.model_selection` и теперь ее импортировать так: ```from sklearn.model_selection import train_test_split```. Переписывать все код под старую библиотеку на новую больно, неприятно и занимает кучу времени. Поэтому можно поднять виртуальную среду, в которую скачать и установить библиотеку sklearn версии 0.15, запускать ноутбук автора там, и в конце, деактивировав виртуальную среду, вернуться к своим версиям установленных библиотек. Более продвинутые программисты для таких целей используют не виртуальную среду, а docker, но это уже совесем другая история.
